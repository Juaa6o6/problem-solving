## 1. 구현 문제란?

### 정의
- 알고리즘보다 **문제를 읽고 그대로 코드화하는 능력**을 보는 유형
- 복잡한 수학이나 자료구조보다 **조건 분기, 반복, 시뮬레이션**이 핵심

### 대표적인 서브 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **시뮬레이션** | 과정을 단계적으로 따라감 | 로봇 이동, 게임 진행 |
| **완전탐색 (Brute Force)** | 모든 경우의 수를 탐색 | 자물쇠 해제, 최솟값 찾기 |
| **격자/그리드 탐색** | 2D 배열에서 이동/탐색 | 미로, 체스판 |
| **문자열 처리** | 문자열 파싱 및 변환 | 암호 해독, 형식 변환 |
| **수학적 구현** | 규칙·공식을 코드로 표현 | 달팽이 배열, 소수 판별 |

## 2. 핵심 접근법

### 풀이 프로세스 5단계

```
1. 문제 읽기      → 입력/출력 형식, 제한 조건 파악
2. 예제 손으로 풀기 → 흐름을 직접 추적해서 로직 이해
3. 설계           → 자료구조 선택, 반복/조건 구조 스케치
4. 코딩           → 엣지 케이스 염두에 두고 구현
5. 검증           → 예제 + 경계값으로 테스트
```

### 구현 문제 풀 때 마음가짐

- "이 문제는 어떤 알고리즘인가?" X
- "문제에서 시키는 대로 차근차근 구현하자" ✅
- 복잡해 보여도 **조건을 잘게 쪼개면** 단순해진다

## 4. 자주 나오는 유형별 패턴

### 4-1. 격자에서 이동 시뮬레이션

```python
def simulate():
    N, M = map(int, input().split())
    grid = [list(input()) for _ in range(N)]
    
    # 시작 위치
    x, y = 0, 0
    
    # 이동 명령
    commands = input().split()
    
    dx = {'U': -1, 'D': 1, 'L': 0, 'R': 0}
    dy = {'U':  0, 'D': 0, 'L': -1, 'R': 1}
    
    for cmd in commands:
        nx = x + dx[cmd]
        ny = y + dy[cmd]
        
        # 경계 체크
        if not (0 <= nx < N and 0 <= ny < M):
            continue
        
        # 조건 체크 (벽 등)
        if grid[nx][ny] == '#':
            continue
        
        x, y = nx, ny
    
    print(x + 1, y + 1)
```

### 4-2. 완전탐색 (모든 경우 탐색)

```python
from itertools import permutations, combinations, product

# 순열: n개 중 r개 선택, 순서 있음
for perm in permutations([1, 2, 3], 2):
    print(perm)  # (1,2), (1,3), (2,1), ...

# 조합: n개 중 r개 선택, 순서 없음
for comb in combinations([1, 2, 3], 2):
    print(comb)  # (1,2), (1,3), (2,3)

# 중복 순열
for prod in product([0, 1], repeat=3):
    print(prod)  # 000, 001, 010, ...

# 직접 완전탐색
def brute_force(arr):
    n = len(arr)
    result = float('inf')
    
    for i in range(n):
        for j in range(i + 1, n):
            result = min(result, arr[i] + arr[j])
    
    return result
```

### 4-3. 달팽이 배열 / 나선형 탐색

```python
def snail(N):
    arr = [[0] * N for _ in range(N)]
    
    # 우 하 좌 상
    dx = [0, 1, 0, -1]
    dy = [1, 0, -1, 0]
    
    x, y, d = 0, 0, 0
    
    for num in range(1, N * N + 1):
        arr[x][y] = num
        nx, ny = x + dx[d], y + dy[d]
        
        # 범위를 벗어나거나 이미 채워진 경우 방향 전환
        if not (0 <= nx < N and 0 <= ny < N) or arr[nx][ny] != 0:
            d = (d + 1) % 4
            nx, ny = x + dx[d], y + dy[d]
        
        x, y = nx, ny
    
    return arr
```

### 4-5. 조건에 따른 상태 변화 (게임/시뮬레이션)

```python
def game_simulate(commands):
    # 상태 정의
    state = {
        'score': 0,
        'position': (0, 0),
        'direction': 0,  # 0=북, 1=동, 2=남, 3=서
        'items': []
    }
    
    for cmd in commands:
        if cmd == 'MOVE':
            # 이동 처리
            pass
        elif cmd == 'TURN_LEFT':
            state['direction'] = (state['direction'] - 1) % 4
        elif cmd == 'TURN_RIGHT':
            state['direction'] = (state['direction'] + 1) % 4
        elif cmd.startswith('PICK'):
            item = cmd.split()[1]
            state['items'].append(item)
    
    return state
```

## 5. 실수 방지 체크리스트

### 자주 하는 실수 TOP 10

```
□ 1. 배열 범위 초과 (IndexError)
     → 이동 전 반드시 범위 체크: 0 <= nx < N and 0 <= ny < M

□ 2. 얕은 복사로 원본 훼손
     → 2D 배열은 copy.deepcopy() 또는 [row[:] for row in grid]

□ 3. 1-indexed vs 0-indexed 혼용
     → 문제 읽을 때 인덱스 기준 명시하기

□ 4. 방향 벡터 순서 오류
     → dx, dy 배열 정의 후 직접 손으로 검증

□ 5. 정수 나눗셈 실수
     → // (몫), % (나머지), / (실수 나눗셈) 구분 철저히

□ 6. 빈 리스트 예외 처리 누락
     → max([]) 등은 ValueError 발생

□ 7. 전역 변수 / 지역 변수 혼용
     → 함수 내에서 전역 변수 수정 시 global 선언

□ 8. 문자열과 정수 비교
     → input()은 항상 문자열 반환 → int() 변환 필요

□ 9. 조건 분기 누락 (else 빠짐)
     → if-elif-elif 구조에서 마지막 else 꼭 확인

□ 10. 반복 종료 조건 오류
      → while 루프 무한루프 주의, 탈출 조건 명확히
```

## 6. 예제 문제 풀이

### 예제 1: 체스판 위의 나이트 이동

**문제**: 8×8 체스판에서 나이트의 현재 위치가 주어질 때, 이동 가능한 경우의 수를 출력하라.

```python
def knight_moves():
    pos = input()  # 예: "a1"
    
    # 열(col)과 행(row) 변환
    col = ord(pos[0]) - ord('a') + 1  # a=1 ~ h=8
    row = int(pos[1])
    
    # 나이트 이동 벡터 (8가지)
    moves = [
        (-2, -1), (-2, 1), (-1, -2), (-1, 2),
        ( 1, -2), ( 1, 2), ( 2, -1), ( 2, 1)
    ]
    
    count = 0
    for dc, dr in moves:
        nc, nr = col + dc, row + dr
        if 1 <= nc <= 8 and 1 <= nr <= 8:
            count += 1
    
    print(count)

knight_moves()
```

### 예제 2: 문자열 재정렬

**문제**: 알파벳과 숫자로 구성된 문자열에서 알파벳은 오름차순, 숫자의 합을 뒤에 붙여 출력하라.

```python
def reorder_string():
    s = input()
    
    letters = sorted([c for c in s if c.isalpha()])
    digit_sum = sum(int(c) for c in s if c.isdigit())
    
    result = ''.join(letters)
    if digit_sum > 0:
        result += str(digit_sum)
    
    print(result)

# 입력: "K1KA5CB7"
# 출력: "ABCKK13"
reorder_string()
```

### 예제 3: 게임 맵 탈출 시뮬레이션

**문제**: N×M 격자에서 'S'에서 출발해 명령에 따라 이동하고, 벽('#')은 통과 불가, 범위 벗어나면 무시. 최종 위치 출력.

```python
def escape_map():
    N, M = map(int, input().split())
    grid = [list(input()) for _ in range(N)]
    
    # 시작 위치 찾기
    sx, sy = 0, 0
    for i in range(N):
        for j in range(M):
            if grid[i][j] == 'S':
                sx, sy = i, j
    
    commands = input().split()
    
    direction_map = {
        'U': (-1, 0), 'D': (1, 0),
        'L': (0, -1), 'R': (0, 1)
    }
    
    x, y = sx, sy
    for cmd in commands:
        if cmd not in direction_map:
            continue
        
        dx, dy = direction_map[cmd]
        nx, ny = x + dx, y + dy
        
        # 범위 및 벽 체크
        if 0 <= nx < N and 0 <= ny < M and grid[nx][ny] != '#':
            x, y = nx, ny
    
    print(x + 1, y + 1)

escape_map()
```

## 7. 시간복잡도 & 최적화 팁

### 제한 시간에 따른 가능한 복잡도

| N의 크기 | 가능한 복잡도 | 접근법 |
|----------|-------------|--------|
| N ≤ 10 | O(N!) | 완전탐색, 순열 |
| N ≤ 20 | O(2^N) | 비트마스크 |
| N ≤ 500 | O(N²) | 이중 반복문 |
| N ≤ 10,000 | O(N log N) | 정렬, 이분탐색 |
| N ≤ 1,000,000 | O(N) | 해시, 누적합 |

### 파이썬 구현 최적화

```python
# 1. 리스트 컴프리헨션 활용 (append보다 빠름)
result = [x * 2 for x in arr if x > 0]

# 2. 집합(set)으로 O(1) 탐색
visited = set()
if (x, y) not in visited:  # O(1)
    visited.add((x, y))

# 3. deque로 O(1) 앞/뒤 삽입
from collections import deque
q = deque([1, 2, 3])
q.appendleft(0)  # 앞에 추가 O(1)
q.popleft()      # 앞에서 제거 O(1)

# 4. Counter로 빈도수 계산
from collections import Counter
freq = Counter("banana")  # {'a': 3, 'n': 2, 'b': 1}

# 5. 누적합으로 구간 합 O(1) 쿼리
arr = [1, 2, 3, 4, 5]
prefix = [0] * (len(arr) + 1)
for i, v in enumerate(arr):
    prefix[i + 1] = prefix[i] + v

# arr[l:r+1]의 합 → prefix[r+1] - prefix[l]
print(prefix[4] - prefix[1])  # arr[1:4]의 합 = 9
```
