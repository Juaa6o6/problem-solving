### 문제 설명

<p>그룹 단어란 단어에 존재하는 모든 문자에 대해서, 각 문자가 연속해서 나타나는 경우만을 말한다. 예를 들면, ccazzzzbb는 c, a, z, b가 모두 연속해서 나타나고, kin도 k, i, n이 연속해서 나타나기 때문에 그룹 단어이지만, aabbbccb는 b가 떨어져서 나타나기 때문에 그룹 단어가 아니다.</p>

<p>단어 N개를 입력으로 받아 그룹 단어의 개수를 출력하는 프로그램을 작성하시오.</p>

### 입력 

 <p>첫째 줄에 단어의 개수 N이 들어온다. N은 100보다 작거나 같은 자연수이다. 둘째 줄부터 N개의 줄에 단어가 들어온다. 단어는 알파벳 소문자로만 되어있고 중복되지 않으며, 길이는 최대 100이다.</p>

### 출력 

 <p>첫째 줄에 그룹 단어의 개수를 출력한다.</p>

---
### 다른 풀이법
```python
N = int(input())
cnt = 0

for _ in range(N):
  word = input()
  if list(word) == sorted(word, key=word.find):
    cnt += 1

print(cnt)
```

### 메모
1. sorted 함수
   
    - `sorted(key=...)`
  
    - 리스트의 요소 자체가 아니라, `key` 함수에 요소를 넣었을 때 나온 결과값(기준값)을 기준으로 정렬함
    - `key=word.find`를 넣으면, 문자 자체가 아닌 '최초 등장 인덱스' 순서대로 문자를 줄 세움
  
2. find 메서드
    
    - `str.find()`
  
    - 문자열에서 특정 문자가 처음 등장하는 인덱스를 반환함
    - 해당 문자가 뒤에 또 나와도, 무조건 맨 처음 위치한 인덱스만 리턴함
    - e.g. `'aba'.find('a')` → 0 → 첫 번째 'a', 마지막 'a' 모두 **0**을 반환
  
3. 판별 원리 (비교)
    - case 1: 그룹 단어일 때 (happy)

      - p가 연속해서 나와도 find 값은 동일하므로 순서가 유지됨
  
      - 최초 등장 인덱스 순으로 나열해도 원본 순서와 일치함
      - sorted 결과: ['h', 'a', 'p', 'p', 'y'] (변화 없음)

    - case 2: 그룹 단어가 아닐 때 (aba)
  
      - 마지막 a는 뒤에 있지만, find 값은 맨 앞 a와 같은 0이다.
  
      - 가운데 b의 find 값은 1이다.
      - 정렬 시 key 값이 작은 a(0)가 b(1)보다 앞으로 이동한다.
      - sorted 결과: ['a', 'a', 'b'] (원본 aba와 다름)
